# Start Here

This section is for the first questions people actually ask:

- "How do I make my first resource?"
- "Should this be detail, collection, or paged?"
- "Can I see a few common resource patterns without reading the deep docs?"

## Reading Order

1. [Your First Resource](./your-first-resource.md)
2. [Choose A Resource Shape](./choose-a-resource-shape.md)
3. [Common Resource Recipes](./common-resource-recipes.md)

## What This Section Assumes

Nothing beyond the shipped public surface.

The examples stay on:

- `signals.api(...)`
- `api.url(...)`
- `.detail(...)`
- `.list(...)`
- `.paged(...)`
- `family.line(...)`

The raw `signals.resource.*(...)` lane is mentioned where it matters, but it is
not the starting point here.
